## M-T6.15 — Feliz silent-drop fallthroughs → implemented / honestly gated — `done` · **S→M** · P1 ⭐ wrong failure mode
Started as the silent-drop → fail-fast conversion (both `default:` arms threw instead of emitting compiling-but-wrong F#); the reachable arms were then **implemented** (Fable-compile-verified). Now on the Feliz MVU action path:
- **Statements** (`feliz/update-emit.ts`, `Cmd`-aware arm assembler): scalar `+=`/`-=`; **collection** `+=`/`-=` (was mis-emitted as list arithmetic `model.Xs + v` — invalid F#; now `xs @ [v]` / `List.filter`); `let`; bare `expression` (`… |> ignore`); `call` to a sibling **action** (`Cmd.ofMsg`, batched) or a ui **function**.
- **Expressions** (`feliz/fs-expr.ts`): predicate `match` (F# `if/elif/else`), single-expr `lambda`, and a bounded **collection/string method set** (`List.contains`/`isEmpty`/`length`; `.ToUpper()`/`.ToLower()`/`.Trim()`/`.Contains()`/`.StartsWith()`/`.EndsWith()`/`.Length`).
- **Stores — now IMPLEMENTED** (the subsystem, not a gate). A `store` folds into the single-program Elmish MVU: each field → a namespaced Model field (`Cart`+`count` → `CartCount`), each action → a Msg case + a store-scoped update arm, store-field reads bind page-view locals (`let count = model.CartCount`), and a store-action call from a page action dispatches (`Cmd.ofMsg (CartAdd …)`). `feliz/index.ts` (fold + `storeWrappers`), `fs-expr.ts` (`storeModelField`/`storeMsgCase` + store-field ref resolution), `update-emit.ts` (store arms), `feliz-target.ts` (`renderStoreFieldRead`/`renderStoreActionCall`). The `loom.feliz-store-unsupported` gate was **removed**. Fable-compile verified; pinned by `test/generator/feliz/store.test.ts`. (Direct `onClick: Cart.clear` — a store action as a handler — silently drops on **every** frontend incl. React/Svelte; a pre-existing cross-frontend gap, not Feliz-specific. The supported idiom `action discard() { Cart.clear() }` works.)
- **Async effects (`match await`) — now IMPLEMENTED** (the renderer, not a gate). A frontend action whose body is `match await <api>.<Agg>.<op>() { <Agg> b => … else => … }` projects to the single-program Elmish MVU as a TRIGGER → RESULT pair: a `<Trigger> of string` Msg carries the route id and fires `Cmd.OfAsync.perform (Api.<op><Agg>Effect id) () <Trigger>Result`; a `<Trigger>Result of Result<<Succ> option, string>` Msg reduces the outcome — the success arm under `(Ok (Some p))` (binding rendered), the `else` under BOTH `(Ok None)` and `(Error _)`. The api fn POSTs `/api/<agg>/<id>/<op>` and decodes the `type`-tagged 200 body (success tag → `Some Decoders.<agg>`, else `None`). Shared shape arbiter `src/ir/util/feliz-async-effect.ts` (`classifyFelizAsyncEffect`) keeps the renderer and the gate in lockstep; `feliz/wire.ts` (`collectPageAsyncEffects` + `renderAsyncEffectFn` + the extra-aggregate decoder), `update-emit.ts` (trigger/result Msg + arms), `index.ts` (collect/fold + the id-passing dispatch wrapper), `fs-expr.ts` (member access now renders the lowercase wire-field name verbatim — the latent `upperFirst` bug the async path first exercised). Fable-compile + runtime-smoke verified (`generated-feliz-build.yml` detail page `Reserve` → `Cmd.OfAsync` → result arm flips page state); pinned by `test/generator/feliz/async-effect.test.ts` + `test/ir/feliz-async-effect.test.ts`. The `loom.feliz-async-effect-unsupported` gate was **narrowed** to the unrendered shapes only (a genuine multi-variant union, an op with params, no `else`, or a host with no route id — a component or non-`:id` page).
- **Defensive fail-fast (unreachable on valid frontend `.ddd`, parity with the JSX `emitStmt`):** `precondition`/`requires`/`emit`/`return`, and value-position `this`/`id`/`action-ref`/`new`.

Net: **zero silent drops and zero codegen crashes on valid `.ddd`** — every reachable construct is implemented or gated at validation. Tests: `test/generator/feliz/action-body.test.ts` (arms) + `async-effect.test.ts` (the `match await` projection), `fail-fast.test.ts` (backstops), `test/ir/store.test.ts` + `feliz-async-effect.test.ts` (the narrowed gate + shape classifier). **Status: DONE** — the Feliz store subsystem (PR #1901) and the `match await` async-effect renderer both landed; each carries a Fable-compile + runtime CI gate.
