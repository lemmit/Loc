# T1 ‚Äî UI & frontend ceiling

*The product's #1 weak spot ([weak-spot review](../audits/architecture-weak-spots-2026-07.md) ¬ß1): the scaffolded UI is excellent until the closed primitive set runs out, then customization is an all-or-nothing ejection. These missions raise the ceiling and soften the cliff.*

## M-T1.1 ‚Äî Paged, sorted, filtered Table ‚Äî `partial` ¬∑ **L** ¬∑ P1
The wire ships the `paged` carrier; no frontend consumes it. `Table`/`QueryView` fetch entire collections; no column sort, no interactive filter. Add pagination/sort/filter to the `Table` primitive across the four JSX walkers + HEEx and the pack templates (server-driven via `paged` + query params where the source supports it, client-side fallback otherwise).
**Decisions (design pass 2026-07-13, maintainer-signed):** (A) route interactive state through page-level `state {}` + the existing `renderStateWrite` seam ‚Äî no new primitive-local-state seam (invariant #7); (B) **client-side v1** (sort/filter/slice the bound array); server-driven `QueryView`-unwrap is a decoupled slice 2 that auto-upgrades once M-T2.6 flips the scaffold's unbounded `.all` to paged; (C) the 4 JSX targets land together, HEEx pinned in `heex-parity` with a reason (LiveView `handle_event` round-trip is a different topology), Feliz attempted / fail-fast per M-T6.15.
**Client-side column SORT ‚Äî landed + build-gate-verified on all 4 JSX targets (PR #1890, slices 1‚Äì4):**
- The optional `WalkerTarget` seams `renderSortableHeader` + `renderSortedRows` (`src/generator/_walker/target.ts`); `Table(sortKey:, sortDir:, Column(..., sortable:, field:))` in `src/generator/_walker/primitives/table.ts`, **gated** so an un-ported target stays byte-identical.
- **React** (`tsxTarget`) ‚Äî inline `[...rows].sort` with `as`-cast comparator. **Vue/Svelte/Angular** ‚Äî a shared `sortRows()` helper (`src/generator/_frontend/table-sort-helper.ts` ‚Üí `src/lib/table-sort.ts`) because their strict templates reject the inline cast; imported via the `usesTableSort` walker flag (Angular re-exposes it as a `protected readonly` member ‚Äî templates can't call free imports). Signals/`$state`/`ref` write idioms per target.
- **Verified end-to-end** (not just structural): React (react tsc), Vue (`vue-tsc` + `vite build`), Svelte (`svelte-check` + `vite build`), Angular (`ng build`) all green on a generated sortable-table project. Tests: `test/generator/{react,vue,svelte,angular}/table-sort.test.ts`.
- **Feliz + HEEx** gracefully degrade to a plain (unsorted) table ‚Äî the seams are absent, so the sort args are ignored (output correct, not broken). Feliz doesn't map cleanly (Elmish `renderStateWrite ‚Üí "()"`; sort needs Set-Msg/update plumbing + Fable verification); HEEx needs a LiveView `handle_event` round-trip. Both pinned as known sort gaps.
- **Scaffold wiring LANDED (slice 5):** `scaffoldList` marks every column `sortable:` with an explicit `field:` and threads `sortKey:`/`sortDir:` into the Table; the List page declares the two string state fields. Scaffold list pages sort out of the box on the 4 JSX frontends. Feliz/HEEx carry the two extra state fields as harmless unused model-fields / mount-assigns (verified: React scaffold `tsc` green, Feliz + Phoenix scaffolds generate).
- **a11y fix (slice 5 follow-up):** the sortable header was a `<span onClick>`, which trips `svelte-check --fail-on-warnings` (the `generated-svelte-build` gate) + axe. Now a real `<button>` (keyboard-focusable, implicit ARIA role, reset chrome) on all 4 JSX targets.
**Client-side PAGINATION ‚Äî landed + build-gate-verified on all 4 JSX targets (slice 6):**
- New **optional** `WalkerTarget` seam `renderPager(spec)` + `PagerSpec` (page ref, `pageSize`, `totalExpr`); `Table(page:, pageSize:)` in `table.ts` slices rows to the active window (built generically from `renderStateRead`) and appends a Prev / "Page N of M" / Next pager. **Gated** so an un-ported target renders unpaged (byte-identical). The pager total derives from the post-sort array (non-null when sort is active) so no redundant `?? []` guard is emitted (a double guard is a strict-Angular TS2869 error).
- **int-typed page state:** generalized `page()`/`stateBlock()` (`StateFieldSpec`) to carry typed fields + literal inits, and taught the **Vue page-shell** to honour `field.init` (Angular/Svelte/React already did). The scaffold list declares a 1-based `pageNum: int` state (named `pageNum`, not `page` ‚Äî reserved keyword, would break `unfold`) and threads `page:`/`pageSize: 10` into the Table.
- **Verified end-to-end:** React (tsc), Vue (`vue-tsc` + `vite build`), Svelte (`svelte-check` 0-warnings + `vite build`), Angular (`ng build`) all green on generated paged projects. Feliz/HEEx degrade (extra `pageNum`/`sortKey` model-fields, verified generate). Tests: `test/generator/{react,vue,svelte,angular}/table-pagination.test.ts`.
**Server-driven PAGINATION + SORT ‚Äî landed (slice 9, coordinated with M-T2.6):** the auto-`findAll` is now paged-by-default (`ensureFindAll` ‚Üí `paged<T>`), so the scaffold list consumes the envelope server-side: `useAll<X>({page, pageSize, sort, dir})` reads `.data.items`, pages off `.data.totalPages`, and sends `?page=&pageSize=&sort=&dir=` to the whitelisted-ORDER-BY route across all five backends. `QueryView paged:` + `Table serverPaged:` gate the unwrap; the client-pager primitive stays valid for non-paged array sources / explicit finds. HEEx unwraps `.items` (sort/pager pinned, heex-parity reason); Feliz decodes the `items` field (Model stays `'T list`, page 1, pager pinned). **Acceptance e2e landed:** `test/behavioral/pagination.mjs` seeds 1000 rows over the real HTTP surface and asserts the paged window/counters/ORDER BY end-to-end (gated in `behavioral-e2e.yml`).
**Client-side FILTER ‚Äî landed on all 4 JSX targets (slice 7):** a `Table(filter: <stateRef>)` renders a search box ABOVE the table (via the optional `renderFilterInput` seam) and wraps the bound rows in a case-insensitive substring match over `Object.values(row)` (via `renderFilteredRows`), applied BEFORE sort/slice so the pager counts the filtered set. **Gated** so an un-ported target renders no box + unfiltered rows (byte-identical). React inlines the `.filter(...)`; Vue/Svelte/Angular call the shared `filterRows` helper added to `src/lib/table-sort.ts` (their strict templates reject the inline `Object.values` cast ‚Äî same split as `sortRows`), imported via the new `usesTableFilter` walker flag (Angular re-exposes it as a `protected readonly` member). Two-way binding per idiom: React controlled `value`/`onChange`, Vue `v-model`, Svelte `bind:value`, Angular `[value]`/`(input)` + `$any`. Client-only (gated off `serverPaged` ‚Äî a server-driven filter is a later slice, like sort/pager's server mode). **Multi-root wrap (found by the local `tsc` build-verify):** the filter box + table are adjacent JSX siblings, illegal in a `QueryView`'s `{cond && (‚Ä¶)}` single-expression slot (`TS2657`). New optional `wrapMultiRoot` seam wraps them in a React `<>‚Ä¶</>` fragment (Vue/Svelte/Angular tolerate multiple template roots ‚Üí omit the seam, byte-identical). **Scoped to the filter box only** ‚Äî the pager alone is emitted under a pack wrapper (`<Paper>`) in scaffolded pages, so wrapping it too would add a redundant fragment there (byte-drift on the page-emitter fixture) for no gain; a bare pager-only `QueryView` stays a pre-existing edge. **Build-verified locally against the real compilers:** React `tsc --noEmit` (filter+sort+page), Vue `vue-tsc`, Svelte `svelte-check --fail-on-warnings` all green; Angular gated on CI (`generated-angular-build` ‚Äî local `ng build` blocked by an env Node patch-version floor, and the change is a no-op for the fragment seam + reuses the ng-build-verified `sortRows` member idiom). Tests: `test/generator/{react,vue,svelte,angular}/table-filter.test.ts` (incl. the filter‚Üísort‚Üíslice composition, pager-counts-filtered, and the fragment-wrap assertion). **Remaining slices:** (8) Feliz/HEEx sort + pagination + filter (Elmish `Set-Msg`/update plumbing + LiveView `handle_event`) if de-pinned; (server-driven filter) folds in once a `QueryView`-level filter param exists. Status stays `partial` for the Feliz/HEEx tail; the client-side core (slices 1‚Äì7, 9) + acceptance e2e are done.
Sources: [pagination-design-note](../old/proposals/pagination-design-note.md), [completeness-audit Tier 1](../audits/completeness-audit-2026-07.md), weak-spots ¬ß1. Touches: `src/generator/_walker/primitives/table.ts`, walker targets, all pack `primitive-table.hbs`.
Acceptance: showcase list pages page/sort against a seeded 1k-row backend; all `generated-*-build` + e2e gates green.

## M-T1.2 ‚Äî `FileUpload` field primitive + `File` type ‚Äî `partial` ¬∑ **L** ¬∑ P1
**Slice 1 shipped** (PR #2007): the `File` field type + its frozen wire contract. `File` is a new leaf `PrimitiveName` (modeled on `json` ‚Äî JSONB-backed, wire-only, no expression semantics) whose wire representation is the fixed object `FileRef = { url, key, contentType, size }` (expanded per-backend like `money`'s `{amount,currency}`). The field type + wire shape + JSONB column land on **all five backends** (forced by the exhaustive `PrimitiveName` switches ‚Äî keeps the wire uniform); the **upload/download endpoints + a dependency-free `localDisk` object store are Hono-only** this slice. IR gate `loom.file-field-needs-object-storage` (a `File` field on a deployable binding no `objectStore` dataSource is an error). Hono emits app-level `POST /files` (multipart ‚Üí mint uuid key ‚Üí store bytes+contentType ‚Üí return `FileRef`) + `GET /files/:key`, gated so a File-free model is byte-identical. Proven by the `LOOM_TS_BUILD` fixture `test/e2e/fixtures/ts-build/file-field.ddd` (generate ‚Üí tsc ‚Üí bundle green). Design: [`missions/M-T1.2-file-field-slice1-design.md`](missions/M-T1.2-file-field-slice1-design.md).
**Remaining (stacked follow-ons):** Slice 2 ‚Äî upload/download endpoints + object-store adapters on the other four backends; Slice 3 ‚Äî `s3` presigned upload (wire the shipping `signedUrl` verb to `File`); **Slice 4 ‚Äî the `FileUpload` input primitive across walkers/packs (the visible UI payoff ‚Äî the original M-T1.2 headline), consuming this frozen wire shape**; File-delete object cleanup (owner-deferred: a deleted row leaves the object).
